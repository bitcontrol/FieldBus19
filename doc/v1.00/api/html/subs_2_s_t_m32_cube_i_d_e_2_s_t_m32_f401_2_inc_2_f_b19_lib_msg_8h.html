Detailed Description

This file publishes the interface of the FieldBus19 message library module.

  • Filename: FB19LibMsg.h
  • Platform(s): All
  • Project: FieldBus19
  • Created: May 17, 2022

  • Description: This file publishes the interface of the FieldBus19 message library module.

    How to use the FB19LibMsg_append...() and FB19LibMsg_remove...() functions effectively:

    • Write data into the messages using the _append() functions
    • Read data in the reverse order it has been written to the messages
    • That way, no indexes that keep track of the writes and reads are required

int FB19LibMsg_appendChar ( FB19Msg_t *  msg,
char  value 
)

Appends an 8 bit character to the payload of the message and increments its payload size by the size of the argument in bytes.

If the data doesn't fit into the message because its end would exceed the end of the payload section, it won't be added and R_ERROR is returned.

Parameters
[in/out]msg points to the message that will store the passed argument when this function has returned successfully. In case of an error, it won't be changed.
[in]valueis the char value to be appended to the message.
Returns
R_SUCCESS if the parameter checks pass and the value could be appended, R_ERROR otherwise.

◆ FB19LibMsg_appendString()

int FB19LibMsg_appendString ( FB19Msg_t *  msg,
const char *  str 
)

Appends the passed string terminated by a '\0' character to the payload of the message including the '\0' character and increments its payload size by the size of the string including its terminating '\0' character.

If the data doesn't fit into the message because its end would exceed the end of the payload section, it won't be added and R_ERROR is returned.

The string is appended in reverse byte order to the payload buffer ('\0' character is inserted first). That way it can be removed from the payload later (see _removeString()) without having to specify its length.

Parameters
[in/out]msg points to the message that will store the passed argument when this function has returned successfully. In case of an error, it won't be changed.
[in]stris the string to be appended to the message.
Returns
R_SUCCESS if the parameter checks pass and the string could be appended, R_ERROR otherwise.

◆ FB19LibMsg_removeData()

int FB19LibMsg_removeData ( FB19Msg_t *  msg,
void *  buffer,
size_t  size 
)

Removes arbitrary data from the end of the payload and decreases its size by the size of the returned data in bytes.

If the data couldn't be removed because its beginning would exceed the beginning of the payload section, it won't be removed and R_ERROR is returned. The caller has to make sure the passed buffer is large enough to store the returned data. This is exactly size number of bytes, or 0 bytes in case of an error.

Parameters
[in]msgpoints to the message from which the data will be removed.
[out]bufferpoints to the variable that will hold the data when this function has returned successfully. In case of an error, it won't be changed.
[in]sizeis the number of bytes that shall be removed from the message. If it doesn't contain the requested amount of payload data, the output variable won't be changed and the function returns R_ERROR.
Returns
R_SUCCESS if the parameter checks pass and the argument could be removed, R_ERROR otherwise.
